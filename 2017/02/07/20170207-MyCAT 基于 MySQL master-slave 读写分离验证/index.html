host=192.168.1.236，port=3306
说明读操作在主库上执行，因为balance=”0”表示不开启读写分离

修改schema.xml dataHost=localHost1的balance=”2”，重启Mycat


从日志可以看出，主库和从库均参与了读操作

修改schema.xml dataHost=localHost1的balance=”3”

从日志可以看出，只有从库参与了读操作
balance=”1”的情形因未配置双主未进行测试
