CSRF 原理

CSRF（Cross-site request forgery），中文名称：跨站请求伪造，也被称为：one click attack/session riding，缩写为：CSRF/XSRF。

可以这么理解CSRF攻击：攻击者盗用了你的身份，以你的名义发送恶意请求。CSRF能够做的事情包括：以你名义发送邮件，发消息，盗取你的账号，甚至于购买商品，虚拟货币转账……造成的问题包括：个人隐私泄露以及财产安全。

受害者只需要完成如下两件事情，攻击者就能够完成 CSRF 攻击：

  • 登录受信任网站A，并在本地生成 Cookie
  • 在不登出 A的情况下，访问危险网站 B

看到这里，你也许会说：“如果我不满足以上两个条件中的一个，我就不会受到CSRF的攻击”。是的，确实如此，但你不能保证以下情况不会发生：

  1. 你不能保证你登录了一个网站后，不再打开一个tab页面并访问另外的网站。
  2. 你不能保证你关闭浏览器了后，你本地的Cookie立刻过期，你上次的会话已经结束。（事实上，关闭浏览器不能结束一个会话，但大多数人都会错误的认为关闭浏览器就等于退出登录/结束会话了……）
  3. 上图中所谓的攻击网站，可能是一个存在其他漏洞的可信任的经常被人访问的网站。

CSRF 防御

  1. 将 Cookie 设置为 HttpOnly

CSRF 攻击很大程度上是利用了浏览器的 Cookie，为了防止站内的 XSS 漏洞盗取 Cookie，设置 HttpOnly 后，避免
了攻击者伪造 Cookie 的情况出现。

  1. 使用 Token

利用 Token 进行客户端校验，直接杜绝使用 Cookie

  1. 通过 Referer 识别

根据 HTTP 协议，在 HTTP 头中有一个字段叫 Referer，记录了该 HTTP 请求的来源地址。可以通过校验 Referer 信息来判断
是否是 CSRF 攻击。
