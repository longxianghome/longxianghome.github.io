标记-清除算法

标记-清除（Mark-Sweep）算法是最基础的收集算法。

首先标记出所有需要回收的对象，在标记完成后统一回收所有被标记的对象。

标记-清除算法的不足：

  1. 标记和清除两个过程的效率都不高
  2. 标记清除后产生大量不连续的内存碎片，碎片过多，导致需要分配大对象时，无法分配，触发再次垃圾收集

复制算法

复制（Copying）收集算法将可用内存按容量划分为大小相等的两块，每次只使用其中的一块。

当一块的内存用完了，就将还存活的对象复制到另一块上面，然后再把已使用过的内存空间一次清理掉。

HotSpot 的 Heap Memory 中 Young Generation 就是按照复制算法的思路进行内存回收的。

HotSpot 假设大部分的对象都是生命短暂的。

HotSpot 将 Young Generation 按 8:1:1 的比例划分为 Eden 空间:From Survivor 空间:To Survivor 空间。

From Survivor 空间 和 To Survivor 空间 实际是两块一模一样的 Survivor 空间。

HotSpot 同时只会使用 Young Generation 的 Eden 空间和 其中一个 Survivor 空间（称为 From Survivor 空间），当回收时将幸存对象复制到另一个 Survivor 空间（称为 To Survivor 空间），然后一次性清除 Eden 空间和 From Survivor 空间。

标记-整理算法

标记-整理算法是针对 Old Generation 的。

标记的过程和标记-清除算法中的标记过程一致，但是后续步骤是让幸存的对象都向一端移动，等于整理了内存空间，是碎片的内存空间变得连续。

分代收集算法

分代收集算法就是根据内存内对象的特点，选择复制算法或者标记-清除算或者标记-整理算法。
