MyBatis XML 配置文件 properties 元素

properties 元素是用于配置属性的，比如 JDBC 连接信息， MyBatis 提供三种配置方式：

  • property 子元素
  • properties 配置文件
  • 代码参数传递

properties 配置文件

properties 配置文件就是将需要的配置信息单独存放于一个 properties 文件中：

property 子元素

代码参数传递

代码参数传递是因为 SqlSessionFactoryBuilder 提供有重载的构造函数支持方法参数传递 properties

优先级

既然 MyBatis 提供了多种配置方式，那么不同的配置方式可能同时存在，并且可能会重复配置，势必需要设定不同配置方式的优先级：

  1. properties 元素中 property 子元素首先被读取
  2. properties 元素中 resource 属性或者 url 属性读取的 properties 文件次之，若出现同名属性则覆盖
  3. SqlSessionFactoryBuilder 构造函数中 properties 对象最后，若出现同名属性则覆盖

故 方法参数传递优先级 > properties 文件传递优先级 > properties 元素 property 子元素优先级

推荐方式

虽然 MyBatis 支持多种配置方式，但是不推荐同时使用，混合使用会提高混乱程度，首选 properties 文件的方式

MyBatis XML 配置文件概述

MyBatis XML 配置文件内容比较多，准备按不同的元素分别分析，这里先做一个概述。

MyBatis XML 配置文件层次结构：

configuration 配置

  • properties 属性
  • settings 设置
  • typeAliases 类型别名
  • typeHandlers 类型处理器
  • objectFactory 对象工厂
  • plugins 插件
  • environments 环境
    • environment 环境变量
    • transactionManager 事务管理器
    • dataSource 数据源
  • databaseIdProvider 数据库厂商标识
  • mappers 映射器

上述配置项中只会分析一些常用，不常用的后续有需要再分析

MyBatis 生命周期

MyBatis 生命周期其实就是几个组件的生命周期。

SqlSessionFactoryBuilder

SqlSessionFactoryBuilder 通过 XML 或者 Java 代码中的配置信息来构建 SqlSessionFactory。

其作用就是构建 SqlSessionFactory，一旦构建了 SqlSessionFactory 即失去存在的意义，所以其生命周期很短暂，等同于方法内局部变量的生命周期。

SqlSessionFactory

SqlSessionFactory 的作用是创建 SqlSession，而 SqlSession 是会话，类似于 JDBC Connection，每一个和数据库交互都需要，所以 SqlSessionFactory 应该存在于整个应用的生命周期。

一个数据源只能对应一个 SqlSessionFactory，SqlSessionFactory 是单例，是单例模式的实践。

SqlSession

SqlSession 是一个会话，只是存在于每一个和数据库交互的过程中，一旦一次交互结束，生命周期就结束了。

需要注意的就是 SqlSession 会话中的 事务和资源关闭。

Mapper

Mapper 只是接口，每当 SqlSession 和数据库交互时都会伴随着 Mapper 实例的创建，其生命周期是和 SqlSession 一致的。
