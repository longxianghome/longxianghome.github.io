Java(JVM) Memory Model

从图中可以看出 JVM Memory 是分为多个部分的。其中 Heap Memory 广义上分为 Young Generation 和 Old Generation

Heap Memory

Heap Memory 是 JVM 虚拟机管理的内存中最大的一块。Heap Memory 是被所有线程共享的一块内存区域，在 JVM 启动时创建。

此内存区域的唯一目的就是存放对象实例。

当前主流的虚拟机的 Heap Memory 空间都是可扩展的，如果在 Heap Memory 中没有内存可供对象实例化，并且空间无法扩展，则
抛出 OutOfMemoryError 异常。

Heap Memory 可以非物理连续空间，只要保证逻辑连续即可。

Young Generation

Young Generation 是存放对象实例的，当对象被实例化后，内存分配在 Young Generation。

Young Generation 又可以分为三个部分：Eden Memory 和 两个 Survivor Memory

当 Young Generation 空间填满之后， garbage collection(垃圾收集) 会进行内存回收，称为 Minor GC

  • 大多数新创建的对象都位于 Eden 内存空间中
  • 当 Eden 空间充满对象时，执行 Minor GC 并将所有幸存者对象移动到其中一个幸存者空间
  • 当幸存者空间满时，Minor GC 会继续检查幸存者对象，并将其移动到另一个幸存者空间
  • 两个幸存者空间始终是有一个为空的
  • 当一个对象连续两次在幸存者空间都幸存，幸存的对象会被移动到 Old Generation 中

Young Generation 的目标就是尽可能快速的收集掉那些生命周期短的对象。

Old Generation

Old Generation 中包含的对象都是生命比较长的和 Young Generation 中幸存的对象。

Old Generation 的垃圾收集称为 Major GC，需要花费更长的时间。

Stop the World Event

所有的垃圾收集都会导致其他线程都被暂停，而这将影响应用程序的响应。

Minor GC 是非常快速的，应用程序受影响较小。

Major GC 比较耗时，垃圾回收时需要检查所有的对象。

Permanent Generation

Permanent Generation 也称为 Non-Heap Memory 。

Method Area

Method Area 是 Permanent Generation 的一部分，用于存放类结构信息，常量、静态变量、即时编译器编译后的代码等数据

Memory Pool

Memory Pool 主要存放 不可变对象，如字符串字面量

Runtime Constant Pool

Runtime Constant Pool 是 Method Area 的一部分

Java Heap Memory Switches

常用的 VM 选项：

VM 选项 含义
-Xms 设置 Heap Memory 的最小空间大小
-Xmx 设置 Heap Memory 的最大空间大小
-XX:NewSize 设置 Heap Memory 的 Young Generation 最小空间大小
-XX:MaxNewSize 设置 Heap Memory 的 Young Generation 最大空间大小
-XX:PermSize 设置 Heap Memory 的 Permanent Generation 最小空间大小
-XX:MaxPermSize 设置 Heap Memory 的 Permanent Generation 最大空间大小
-Xss 设置每个线程的堆栈大小

Young Generation 中 Eden : From Survivor : To Survivor 比例 8:1:1

Old Generation 大小 = Heap Memory 大小 - Young Generation 大小
