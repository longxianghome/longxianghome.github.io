Spring Framework Bean 生命周期详解

发表于 2016-11-20 | 分类于 Spring Framework |
字数统计: 341 | 阅读时长 ≈ 1

参考 Spring Bean Life Cycle Tutorial https://www.concretepage.com/spring/spring-bean-life-cycle-tutorial

Spring Bean 生命周期：

  1. IoC 容器调用 Bean 的 constructor 方法对 Bean 进行实例化
  2. IoC 根据依赖注入调用 setter 方法
  3. 如果 Bean 实现了 BeanNameAware 接口，回调该接口的 setBeanName() 方法，传入 Bean 的 id，Bean 将持有自己在配置信息中配置的 Bean id
  4. 如果 Bean 实现了 BeanFactoryAware 接口，回调该接口的 setBeanFactory() 方法，传入 Bean 的 BeanFactory，Bean 将持有自己的 BeanFactory
  5. 如果 Bean 实现了 ApplicationContextAware 接口，回调该接口的 setApplicationContext() 方法，传入 Bean 的 ApplicationContext，Bean 将持有自己所在的 ApplicationContext
  6. 如果 Bean 实现了 BeanPostProcessor 接口，回调该接口的 postProcessBeforeInitialzation() 方法
  7. 如果 Bean 实现了 InitializingBean 接口，回调该接口的 afterPropertiesSet() 方法
  8. 如果 Bean 配置了 init-method 方法，则执行 init-method 配置的方法
  9. 如果 Bean 实现了 BeanPostProcessor 接口，回调该接口的 postProcessAfterInitialzation() 方法
  10. Bean 创建完毕，可以使用
  11. Ioc 容器关闭后，如果 Bean 实现了 DisposableBean 接口，回调该接口的 destroy() 方法
  12. 如果 Bean 配置了 destroy-method 方法，则执行 destroy-method 配置的方法

一般通过 BeanPostProcessor 接口、InitializingBean 接口 影响 Bean

*Aware 接口是用于 Bean 初始化时获取 Spring 中的一些对象，如 ApplicationContext

BeanPostProcessor 接口又称为增强处理器
