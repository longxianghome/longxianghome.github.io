MyBatis 二级缓存概述

发表于 2016-07-14 | 分类于 MyBatis 原理 |
字数统计: 386 | 阅读时长 ≈ 2

MyBatis settings 元素中有个一个配置项：cacheEnabled=true 表示启用 MyBatis 缓存，这里的缓存其实是二级缓存。

MyBatis settings cacheEnabled 配置信息都在 Configuration 里 ：

默认选择 Simle类型的 Executor，可以根据配置的 executorType 选择使用 BatchExecutor 或 ReuseExecutor。

而如果配置的 cacheEnabled，最终将使用 CachingExecutor 。

CachingExecutor 是 Executor 一个直接的实现，而 BatchExecutor、ReuseExecutor、SimpleExecutor 都是通过继承 BaseExecutor 间接实现的 Executor。

CachingExecutor 是装饰模式的一种体现：

需要持有 BatchExecutor、ReuseExecutor、SimpleExecutor 中的一种。

MyBatis 二级缓存有两种使用方式：

  • 为每一个 Mapper 分配一个 Cache ，使用 cache 配置
  • 多个 Mapper 共用一个 Cache，使用 cache-ref 配置

MyBatis 二级缓存生效条件：

  1. 二级缓存配置总开关需要配置：cacheEnabled = true
  2. select 语句所在的 Mapper 配置了 cache 或 cache-ref，并且有效
  3. select 语句配置 useCache = true

MyBatis 支持三种二级缓存：

  1. MyBatis 自身默认提供的二级缓存实现：LRU、FIFO、Scheduled 等等
  2. 自定义实现 Cache 接口的缓存
  3. 第三方内存缓存库集成：EHCache 等
