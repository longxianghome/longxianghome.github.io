MyBatis 架构概述

在上一篇 MyBatis SqlSessionFactory 构建过程中发现分析起来比较费劲，所以先来整体了解一下 MyBatis 架构。

MyBatis 架构如上图所示。

Mybatis的功能架构分为三层：

  • API接口层：提供给外部使用的接口API，开发人员通过这些本地API来操纵数据库。接口层一接收到调用请求就会调用数据处理层来完成具体的数据处理。
  • 数据处理层：负责具体的SQL查找、SQL解析、SQL执行和执行结果映射处理等。它主要的目的是根据调用的请求完成一次数据库操作。
  • 基础支撑层：负责最基础的功能支撑，包括连接管理、事务管理、配置加载和缓存处理，这些都是共用的东西，将他们抽取出来作为最基础的组件。为上层的数据处理层提供最基础的支撑。

基础支撑层

  • 反射模块：MyBatis 专门提供了反射模块，封装了 Java 原生的反射，提供更加简洁的 API ，优化了反射操作的性能
  • 类型转换模块：类型转换一方面是实现 Java 数据类型和数据库数据类型的转换；另一方面是实现别名的转换
  • 日志模块：集成框架所需的日志，便于集成第三方日志框架
  • 解析模块：解析配置文件以及 SQL 文件中的占位符解析
  • 资源加载模块：优化类加载器，用于加载相关的类
  • 数据源模块：支持扩展第三方实现，管理数据源
  • 事务管理：抽象了事务处理
  • 缓存模块：MyBatis 支持一级缓存和二级缓存
