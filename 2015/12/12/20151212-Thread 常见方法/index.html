Thread 常见方法

发表于 2015-12-12 | 分类于 Java Thread |
字数统计: 509 | 阅读时长 ≈ 2

简单方法

currentThread()

在上一节中，MyRunnable 中使用了 Thread.currentThread().getName() 这么一个方法，其实获取当前线程的名称。

Thread.currentThread() 是一个静态的 native 方法，用于返回代码段正在被哪个线程调用的信息。

isAlive()

isAlive() 方法是用于判断当前的线程是否处于活动状态

sleep()

sleep() 方法是让当前线程正在执行的线程暂停执行，需要设置一个毫秒为单位的时间

getName()

getName() 方法是获取线程的名称

getId()

getId() 方法是获取线程的唯一标识

yield()

yield() 方法是让当前线程放弃 CPU 资源，将它让给其他线程去占用，但是放弃的时间是不确定的，而且可能放弃了又立即获取了

start()

start() 方法是线程启动的唯一方法，需要和 run() 方法做区分，run() 方法只是 Thread 中一个普通的方法，并不能让线程启动

暂停线程

suspend() 方法可以暂停线程执行
resume() 方法可以恢复线程执行

不过这两个方法都将独自线程资源，一旦线程调用了 suspend() 方法，必须自己调用 resume() 方法才能恢复线程，容易导致死锁

停止线程

线程的停止比较复杂，Java 提供了三种方法用于线程的停止：

  1. 使用退出标志，使线程可以正常退出
  2. 使用 stop() 方法，但是已经使废弃的方法了
  3. 使用 interrupt() 方法

interrupt() 方法实际只会标记该线程需要停止，并不是立即停止

线程优先级

Java 为线程提供了 setPriority() 方法用于设置线程的优先级，优先级只是一定几率上提高其优先执行，但并不绝对。

Java 将线程的优先级分为 1-10 共 10 个等级。

Java 线程的优先级是可以继承的，如果 A 线程启动了 B 线程，那么 B 线程的优先级与 A 是一样的。
